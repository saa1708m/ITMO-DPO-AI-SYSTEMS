- Воспроизводимый эксперимент ──→ Модуль 2 ──→ ML Engineer ──→ ЛР: MLflow + Registry

- Развёртывание ML-сервиса ──→ Модуль 4 ──→ MLOps Engineer ──→ ЛР: Docker + serving
- Эксплуатация и мониторинг ──→ Модуль 5 ──→ MLOps Engineer ──→ ЛР: CI/CD + monitoring, проект

## 🧩 Как кейсы  встраиваются в дисциплину
- Кейсы могут использоваться как основа для индивидуальных и командных проектных работ бакалавриата.
- При выборе кейса приоритет отдаётся тем сценариям, где можно одновременно оценить экспериментальную часть, архитектуру решения и инженерные практики развёртывания и сопровождения.
- Один и тот же кейс может быть интерпретирован через разные роли КРМ: например, ИИ-ассистент — через ML Engineer, а облачное решение для платформы Сфера — через AI Architect и MLOps Engineer.
- В рамках ФОС кейсы используются только как прикладной контекст для оценки LC-2, LC-3 и LC-5.
