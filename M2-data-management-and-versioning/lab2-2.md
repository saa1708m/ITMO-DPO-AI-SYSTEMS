# Лабораторная работа 2.2. Версионирование и ETL

## Назначение

Освоить инструменты версионирования данных, построить ETL-пайплайн и организовать автоматические проверки качества данных.

## Проверяемые результаты обучения

Студент должен уметь:

- настраивать DVC и Git LFS для управления версиями данных;
- разрабатывать ETL-пайплайн с использованием Python;
- организовывать хранение признаков в виде feature store или его упрощённого аналога;
- создавать автоматические проверки качества данных;
- обеспечивать воспроизводимость пайплайна.

## Постановка задания

1. Создайте Git-репозиторий и настройте Git LFS для больших файлов.
2. Инициализируйте DVC и добавьте под контроль версий набор данных.
3. Реализуйте ETL-пайплайн, который:
   - читает исходные данные;
   - выполняет очистку и преобразование;
   - формирует набор признаков;
   - сохраняет результат в отдельный артефакт.
4. Организуйте хранение признаков:
   - минимальный вариант — отдельный файл или таблица с версионированием;
   - продвинутый вариант — использование Feast или аналогичного инструмента.
5. Реализуйте автоматические проверки качества данных с помощью `Great Expectations`, `pytest` или эквивалентного подхода.
6. Добавьте средство воспроизводимого запуска: `dvc.yaml`, `Makefile` или аналогичный сценарий.
7. Подготовьте README с описанием структуры проекта и команд запуска.

## Формат сдачи

Обязательные артефакты:

- Git-репозиторий со структурой вида:
  - `data/raw/` — сырые данные;
  - `data/processed/` — обработанные данные;
  - `src/` — код ETL-пайплайна;
  - `tests/` — проверки качества данных;
  - `dvc.yaml` или `Makefile` — сценарий запуска;
  - `.gitattributes` — настройка Git LFS;
  - `README.md` — инструкция по воспроизведению.
- Скриншоты или лог успешного запуска пайплайна и проверок.

## Критерии оценивания

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Настройка DVC и Git LFS | Не настроены | Настроен только один инструмент или есть существенные ошибки | Оба инструмента настроены и используются корректно |
| Реализация ETL-пайплайна | Пайплайн отсутствует или нерабочий | Пайплайн частично работает | Есть полноценный ETL-пайплайн |
| Организация хранения признаков | Не реализована | Реализована частично | Есть корректная организация хранения признаков |
| Автоматические проверки качества данных | Не реализованы | Есть отдельные проверки | Есть автоматизированные проверки, встроенные в процесс |
| Воспроизводимость и документация | Нет инструкций | Инструкции неполные | Есть полные инструкции и воспроизводимый запуск |

## Методические примечания

- Для упрощённого варианта допускается использовать локальный DVC remote.
- Если полноценный feature store настроить сложно, допускается реализовать его упрощённый эквивалент с версионируемым хранилищем признаков.
- Важно показать не только код, но и воспроизводимость процесса.
- Зависимости рекомендуется фиксировать в `requirements.txt` или `pyproject.toml`.
