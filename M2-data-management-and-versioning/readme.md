## 3. Методические указания по выполнению и оцениванию заданий

### 3.1. Общая характеристика заданий

Оценочные средства модуля направлены на проверку способности студента работать с данными в ML-проектах: проводить EDA, контролировать качество данных, организовывать версионирование датасетов и пайплайнов подготовки признаков, обеспечивать воспроизводимость подготовки данных.

- **Тесты** проверяют понимание ключевых понятий и инструментов.
- **Лабораторные работы** проверяют практическое применение этих подходов на реальных или учебных данных.

### 3.2. Методические указания для студента

- Перед выполнением заданий повторите понятия EDA, data quality, DVC, Git LFS, ETL/ELT, feature store.
- В лабораторной работе 2.1 уделите внимание не только построению графиков, но и интерпретации результатов.
- В лабораторной работе 2.2 обеспечьте воспроизводимость: зафиксируйте структуру проекта, зависимости, команды запуска и проверки.
- Все артефакты рекомендуется оформлять в Git-репозитории с понятной структурой и README.

### 3.3. Методические указания для преподавателя

- При проверке тестов обращайте внимание на понимание различий между близкими понятиями: DVC и Git LFS, ETL и ELT, feature store и обычное хранилище данных.
- При проверке лабораторных оценивайте не только работоспособность, но и осмысленность выбора инструментов, качество документации и воспроизводимость.
- Рекомендуется использовать критерии оценивания и вопросы к защите, приведённые в соответствующих КИМ.

## 5. Система оценивания

### 5.1. Шкала для лабораторных работ

Каждая лабораторная работа оценивается по 10-балльной шкале на основе пяти критериев по 0–2 балла каждый.

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Корректность реализации | Задание не выполнено или содержит грубые ошибки | Реализация частично работоспособна | Реализация полностью корректна и соответствует постановке |
| Обоснованность решений | Обоснование отсутствует | Обоснование частичное | Решения аргументированы и связаны с задачей |
| Качество документации | Документация отсутствует или неполна | Документация есть, но неполная | Есть понятная и полная документация по выполнению |
| Воспроизводимость | Результаты нельзя воспроизвести | Воспроизводимость частичная | Есть полные инструкции, зависимости и шаги запуска |
| Анализ и выводы | Выводы отсутствуют | Выводы поверхностны | Есть содержательные выводы и интерпретация |

**Итоговый балл** — сумма по пяти критериям, максимум 10.

### 5.2. Шкала перевода баллов для лабораторных работ

| Баллы | Оценка |
|-------|--------|
| 9–10 | Отлично |
| 7–8 | Хорошо |
| 5–6 | Удовлетворительно |
| 0–4 | Неудовлетворительно |

### 5.3. Шкала для тестов

Каждый правильный ответ оценивается в 1 балл. Максимум — 10 баллов.

| Баллы | Оценка |
|-------|--------|
| 9–10 | Отлично |
| 7–8 | Хорошо |
| 5–6 | Удовлетворительно |
| 0–4 | Неудовлетворительно |

### 5.4. Вес оценочных средств в модуле

- Тест 2.1 — 10%
- Тест 2.2 — 10%
- Лабораторная работа 2.1 — 40%
- Лабораторная работа 2.2 — 40%

Итоговый процент за модуль рассчитывается по формуле:

\[
P = (0.1 \cdot T_1/10 + 0.1 \cdot T_2/10 + 0.4 \cdot L_1/10 + 0.4 \cdot L_2/10) \cdot 100\%
\]

**Порог успешности** — 60%.
