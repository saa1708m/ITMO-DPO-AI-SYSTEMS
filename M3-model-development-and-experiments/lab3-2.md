# Лабораторная работа 3.2. Registry и промоутирование модели

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Лабораторная работа |
| **Модуль** | 3. Разработка, эксперименты и версионирование моделей |
| **Тема** | Реестры моделей и AutoML |
| **Компетенция** | LC-2, LC-5 |
| **Индикатор** | LC-2.3, LC-5.2 |
| **Уровень** | С (средний) |
| **Режим выполнения** | Индивидуально |
| **Время** | 2 часа |

## Назначение

Освоить регистрацию модели в MLflow Model Registry, управление стадиями жизненного цикла модели и оформление model card.

## Проверяемые результаты обучения

Студент должен уметь:

- регистрировать модель в реестре;
- управлять версиями и стадиями модели;
- оформлять model card;
- загружать модель из реестра для инференса;
- обеспечивать воспроизводимость сценария регистрации и использования модели.

## Постановка задания

1. Возьмите лучшую модель из лабораторной работы 3.1.
2. Зарегистрируйте модель в MLflow Model Registry под осмысленным именем.
3. Переведите модель в стадию `Staging`.
4. Проведите дополнительную проверку качества и при соблюдении критериев переведите модель в стадию `Production`.
5. Создайте файл `model_card.md`, включающий:
   - название и версию модели;
   - описание задачи;
   - сведения о данных;
   - ключевые метрики качества;
   - ограничения и риски использования;
   - ссылку на запуск в MLflow.
6. Реализуйте скрипт или notebook, который загружает production-модель из реестра и выполняет инференс.
7. Подготовьте README с описанием шагов регистрации, смены стадий и воспроизведения результата.

## Формат сдачи

Обязательные артефакты:

- код регистрации модели и инференса;
- `model_card.md`;
- `README.md`;
- скриншоты или экспорт интерфейса MLflow Model Registry;
- инструкции по воспроизведению.

Ссылка на Git-репозиторий предоставляется преподавателю.

## Критерии оценивания

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| Регистрация модели в реестре | Не выполнена | Выполнена частично | Модель корректно зарегистрирована |
| Управление стадиями | Стадии не использованы | Использована одна стадия без обоснования | Есть логичный переход между стадиями с объяснением |
| Наличие и качество model card | Model card отсутствует | Model card неполная | Есть полная и содержательная model card |
| Загрузка модели из реестра и инференс | Не реализованы | Реализованы частично | Модель загружается из реестра и выполняет инференс |
| Воспроизводимость и документация | Инструкции отсутствуют | Инструкции неполные | Есть полные инструкции и понятная документация |

## Методические примечания

- Для работы можно использовать локальный MLflow server.
- Желательно показать, что выбранная production-модель действительно связана с результатами лабораторной 3.1.
- В model card важно отражать не только достоинства модели, но и ограничения её применения.
