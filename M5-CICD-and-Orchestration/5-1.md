# Тест 5.1. CI/CD для ML

## Цель

Проверить понимание основных принципов CI/CD в ML-проектах, структуры типичного конвейера, различий между DevOps и MLOps, а также назначения линтеров, тестов, сборки и публикации Docker-образов.

## Инструкция

Выберите один правильный ответ в каждом задании.

## Задания

1. Какое из утверждений **наиболее точно** отражает отличие MLOps от классического DevOps?
   а) MLOps используется только для фронтенд-разработки.
   б) MLOps учитывает не только код приложения, но и данные, модели, качество предсказаний и цикл переобучения.
   в) MLOps не предполагает автоматизацию.
   г) MLOps полностью исключает контейнеризацию.
   д) MLOps относится только к управлению требованиями.

2. Какая последовательность шагов наиболее типична для базового CI-конвейера ML-сервиса?
   а) Публикация образа → линтинг → тестирование → сборка.
   б) Линтинг → тестирование → сборка образа → публикация артефакта.
   в) Обучение модели → A/B-тестирование → аудит безопасности.
   г) Оркестрация DAG → сборка образа → настройка RBAC.
   д) Мониторинг → анализ дрейфа → деплой.

3. Для чего в CI-конвейере обычно используются линтеры (`ruff`, `flake8`)?
   а) Для сжатия Docker-образа.
   б) Для автоматического улучшения метрик модели.
   в) Для проверки качества оформления и синтаксической корректности кода.
   г) Для хранения данных обучения.
   д) Для публикации артефактов в registry.

4. Что является типичным результатом шага `build` в CI-конвейере ML-сервиса?
   а) Готовый dashboard в Grafana.
   б) Собранный Docker-образ с приложением и зависимостями.
   в) Размеченный датасет.
   г) Архитектурная диаграмма.
   д) DAG в Airflow.

5. Какой из инструментов чаще всего используется для модульного тестирования Python-кода в ML-проекте?
   а) pytest
   б) Grafana
   в) MinIO
   г) Kubernetes
   д) Vault

6. Зачем публиковать Docker-образ в registry?
   а) Чтобы хранить результаты опросов студентов.
   б) Чтобы использовать образ в следующих этапах доставки и развёртывания.
   в) Чтобы заменить систему контроля версий.
   г) Чтобы запускать линтеры.
   д) Чтобы автоматически переобучать модель.

7. Какой вариант тегирования Docker-образов является более осмысленным в CI?
   а) Использовать только тег `temp`.
   б) Не использовать теги вообще.
   в) Использовать тег по commit SHA и отдельный тег для основной ветки.
   г) Каждый раз случайно генерировать имя вручную.
   д) Всегда использовать только `draft`.

8. Что из перечисленного лучше всего относится к проверкам качества ML-системы в CI?
   а) Проверка схемы данных, unit-тесты и smoke-тест контейнера.
   б) Только ручная защита проекта.
   в) Только презентация для заказчика.
   г) Только запуск ноутбука Jupyter.
   д) Только публикация README.

9. Какова основная цель этапа Continuous Integration?
   а) Отложить проверку кода до экзамена.
   б) Регулярно автоматически проверять изменения при их внесении в репозиторий.
   в) Исключить тестирование.
   г) Заменить разработчиков.
   д) Отказаться от ветвления в Git.

10. Что является хорошей практикой для CI в ML-проекте?
   а) Хранить секреты прямо в файле workflow.
   б) Пропускать тесты ради ускорения сборки.
   в) Автоматизировать повторяемые проверки и минимизировать ручные операции.
   г) Публиковать каждый эксперимент сразу в production.
   д) Не фиксировать версии зависимостей.
