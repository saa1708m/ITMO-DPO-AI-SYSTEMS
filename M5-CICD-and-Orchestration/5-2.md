# Тест 5.2. Оркестрация ML-пайплайнов

## Цель

Проверить понимание принципов оркестрации ML-процессов, DAG-подхода, ролей основных компонентов Apache Airflow и логики автоматического переобучения моделей.

## Инструкция

Выберите один правильный ответ в каждом задании.

## Задания

1. Что такое DAG в контексте Airflow?<br><br>
   а) Хранилище Docker-образов.<br><br>
   б) Ориентированный ациклический граф задач и зависимостей между ними.<br><br>
   в) Формат хранения датасета.<br><br>
   г) Способ контейнеризации Python-кода.<br><br>
   д) Модель классификации.<br><br>

2. Какова основная роль Apache Airflow в ML-проекте?<br><br>
   а) Создание интерфейсов визуализации.<br><br>
   б) Оркестрация и планирование последовательностей задач в пайплайне.<br><br>
   в) Замена Git и систем контроля версий.<br><br>
   г) Обучение нейронных сетей без кода.<br><br>
   д) Создание model card.<br><br>

3. Что обычно понимается под `task` в Airflow?<br><br>
   а) Весь курс целиком.<br><br>
   б) Отдельный исполняемый шаг внутри DAG.<br><br>
   в) Образ Docker Hub.<br><br>
   г) Ветка Git.<br><br>
   д) Набор ролей в команде.<br><br>

4. Что делает scheduler в Airflow?<br><br>
   а) Выполняет линтинг Python-кода.<br><br>
   б) Определяет, какие задачи и когда должны быть поставлены на выполнение.<br><br>
   в) Публикует контейнеры в registry.<br><br>
   г) Проверяет fairness модели.<br><br>
   д) Создаёт README.md.<br><br>

5. Какой сценарий наиболее типичен для auto-retraining?<br><br>
   а) Модель переобучается вручную только по требованию комиссии.<br><br>
   б) Модель переобучается при наступлении триггера: новое окно данных, расписание или деградация качества.<br><br>
   в) Модель переобучается после каждого открытия ноутбука.<br><br>
   г) Модель никогда не переобучается.<br><br>
   д) Модель заменяется случайным образом.<br><br>

6. Какой набор шагов наиболее характерен для DAG переобучения модели?<br><br>
   а) Загрузка данных → обучение → оценка → принятие решения о публикации/обновлении модели.<br><br>
   б) Написание презентации → проведение опроса → публикация статьи.<br><br>
   в) Сборка сайта → публикация баннера → отправка письма.<br><br>
   г) Создание датасета → удаление модели → остановка сервера.<br><br>
   д) Сбор логов → дизайн интерфейса → регистрация домена.<br><br>

7. Для чего в Airflow может использоваться XCom?<br><br>
   а) Для обмена небольшими данными между задачами.<br><br>
   б) Для замены Docker Compose.<br><br>
   в) Для хранения больших бинарных артефактов моделей.<br><br>
   г) Для балансировки сетевого трафика.<br><br>
   д) Для анализа fairness.<br><br>

8. Что является важным риском при автоматическом retraining?<br><br>
   а) Полное отсутствие запуска задач.<br><br>
   б) Продвижение новой модели без проверки качества по целевой метрике.<br><br>
   в) Использование Python в пайплайне.<br><br>
   г) Наличие логирования.<br><br>
   д) Наличие DAG-файла.<br><br>

9. Какой подход является более корректным при обновлении модели в пайплайне?<br><br>
   а) Обновлять production-модель всегда, независимо от качества.<br><br>
   б) Сравнивать новую модель с текущей по заданной метрике и обновлять только при соблюдении критерия.<br><br>
   в) Выбирать модель случайным образом.<br><br>
   г) Полностью отказаться от проверки метрик.<br><br>
   д) Обновлять модель только по названию файла.<br><br>

10. Что лучше всего показывает зрелый подход к оркестрации ML-пайплайна?<br><br>
   а) Все действия выполняются вручную в ноутбуке.<br><br>
   б) Шаги формализованы, зависимости заданы явно, а правила запуска и обновления воспроизводимы.<br><br>
   в) Все файлы лежат в одной папке без структуры.<br><br>
   г) Решения не документируются.<br><br>
   д) Ошибки выполнения не отслеживаются.<br><br>
