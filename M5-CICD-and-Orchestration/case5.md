# Кейс 5. Проектирование CI/CD и оркестрации ML-процесса

## Цель

Проверить способность обучающегося спроектировать базовый CI/CD-конвейер и схему оркестрации ML-процесса для прикладного сервиса машинного обучения.

## Формулировка кейса

Команда разрабатывает ML-сервис для бинарной классификации обращений клиентов. Сервис реализован как REST API на Python. Модель должна регулярно обновляться по новым данным, поступающим ежедневно. Требуется спроектировать минимально жизнеспособный инженерный контур, который позволит:

- автоматически проверять код при каждом изменении;
- автоматически собирать и публиковать контейнерный образ;
- периодически запускать переобучение модели;
- оценивать качество новой модели;
- обновлять рабочую модель только при выполнении заданного критерия качества.

## Задание

Подготовьте структурированный ответ, в котором:

1. Опишите базовый CI-конвейер для данного проекта.
2. Перечислите этапы, которые должны быть автоматизированы.
3. Укажите, какие проверки нужно встроить в CI.
4. Опишите структуру DAG-пайплайна переобучения в Airflow.
5. Поясните, по какому правилу новая модель может быть переведена в staging или production.
6. Назовите 2–3 риска плохо спроектированного пайплайна и способы их снижения.

## Ожидаемые элементы ответа

В ответе ожидаются:

- указание шагов lint → test → build → push image;
- понимание роли registry;
- описание DAG со стадиями загрузки данных, обучения, оценки, decision gate и публикации артефакта;
- указание метрики качества или критерия сравнения новой и текущей модели;
- понимание рисков: публикация некачественной модели, отсутствие воспроизводимости, неявные зависимости, ручные операции.

## Критерии оценивания

### Отлично

Ответ полный, логичный, инженерно состоятельный. Описаны и CI/CD-часть, и оркестрация retraining, и правило принятия решений. Показано понимание рисков и способов их снижения.

### Хорошо

Ответ в целом корректный, но отдельные элементы раскрыты кратко или без достаточной детализации. Логика пайплайна в основном прослеживается.

### Удовлетворительно

Ответ отражает только общую идею процесса, но не показывает чёткой структуры пайплайна или явной логики принятия решений по обновлению модели.

### Неудовлетворительно

Ответ фрагментарный, отсутствует понимание структуры CI/CD и DAG-оркестрации, предложенное решение не позволяет воспроизводимо поддерживать ML-сервис.
