# Тест 7.1. Наблюдаемость и метрики ML-систем

## Паспорт КИМ

| Параметр | Значение |
|---|---|
| Вид КИМ | Тест |
| Модуль | 7 |
| Тема | Наблюдаемость и метрики ML-систем |
| Компетенция | LC-5 |
| Индикатор | LC-5.4 |
| Уровень | Б |
| Время выполнения | 30–40 минут |
| Максимальный балл | 10 |

## Назначение

КИМ предназначен для текущего контроля по теме наблюдаемости ML-систем. Тест позволяет проверить базовое понимание роли мониторинга, различий между типами метрик и назначения основных инструментов наблюдаемости.

## Проверяемые результаты обучения

- различает инфраструктурные, сервисные и модельные метрики;
- понимает назначение Prometheus, Grafana, ELK и централизованного логирования;
- умеет соотносить источник сигнала и тип эксплуатационной проблемы.

## Инструкция студенту

В каждом задании выберите один правильный ответ из пяти предложенных вариантов. Исправления допускаются только до момента сдачи работы. Использование материалов определяется преподавателем.

## Задания

### Вопрос 1
Какой показатель относится прежде всего к сервисным метрикам ML-сервиса?

- A. Заполненность диска на узле
- B. Время ответа API инференса
- C. Размер обучающей выборки

- E. Число признаков в датасете

### Вопрос 2

- A. Хранение датасетов
- B. Сбор и хранение временных рядов метрик
- C. Версионирование моделей и артефактов
- D. Генерация признаков
- E. Публикация Docker-образов

### Вопрос 3

- A. Grafana

- C. Git LFS
- D. Docker Compose
- E. MLflow Registry

### Вопрос 4
Какой вариант относится к модельным метрикам в продуктивной среде?

- A. CPU usage контейнера
- B. Частота перезапуска пода
- C. Latency REST API
- D. Текущая точность, precision, recall или proxy-метрики качества модели
- E. Объём занятой оперативной памяти

- A. Для резервного копирования датасетов

### Вопрос 7

### Вопрос 8

- A. Дашборд должен содержать все доступные метрики без отбора
- B. На дашборде достаточно одной модельной метрики

- A. ELK / OpenSearch Stack

- D. DVC
- E. Terraform

### Вопрос 10
