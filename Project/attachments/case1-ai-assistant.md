# Кейс №1. Разработка интеллектуального ИИ-ассистента для обработки обращений клиентов

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Курсовой проект (кейс) |
| **Модули** | 1–8 (полный цикл) |
| **Компетенции** | LC-2, LC-3, LC-5 |
| **Индикаторы** | LC-2.1–2.3, LC-3.1–3.2, LC-5.1–5.5 |
| **Уровень** | С–П (средний–продвинутый) |
| **Режим выполнения** | Команда 2–3 человека |
| **Срок выполнения** | 4 недели |
| **Максимальный балл** | 10 |

## Назначение

Спроектировать и реализовать сквозное ИИ-решение для автоматической обработки обращений клиентов, включающее классификацию запросов, генерацию ответов и полный MLOps-цикл. Проект моделирует реальную задачу, решаемую Т1 при разработке ИИ-ассистентов для крупных заказчиков (ВЭБ.РФ, ДОМ.РФ).

## Проверяемые результаты обучения

Студент должен уметь:

- формулировать бизнес-задачу как ML-задачу;
- проводить EDA и подготавливать данные для обучения;
- реализовывать и сравнивать модели классификации и генерации;
- организовывать трекинг экспериментов в MLflow;
- контейнеризировать ML-сервисы;
- настраивать CI/CD и оркестрацию пайплайнов;
- развёртывать масштабируемый REST API;
- организовывать мониторинг дрейфа данных и моделей;
- обеспечивать безопасность и governance системы.

## Постановка задания

### 1. Анализ и подготовка данных (Модуль 2)

**Задание:**
- Выберите датасет с обращениями клиентов (например, из открытых источников или синтезируйте самостоятельно).
- Проведите EDA: распределение классов, длина текстов, ключевые слова.
- Очистите данные: удалите дубликаты, обработайте пропуски, выполните токенизацию и нормализацию.
- Настройте DVC для версионирования данных.
- Подготовьте данные для обучения моделей классификации и генерации.

**Ожидаемый результат:**
- Jupyter Notebook с EDA.
- DVC-репозиторий с данными.
- Файл `data/processed/` с очищенными данными.

### 2. Разработка моделей (Модуль 3)

**Задание:**
- Реализуйте как минимум 3 модели для классификации обращений (например, логистическая регрессия, XGBoost, Fine-tuned BERT).
- Реализуйте модель генерации ответов (можно использовать open-source LLM или API).
- Организуйте эксперименты в MLflow Tracking: логируйте параметры, метрики (accuracy, F1, precision, recall), артефакты.
- Сравните модели и выберите лучшую для каждого этапа.
- Зарегистрируйте лучшую модель в Model Registry, переведите в Staging и Production.

**Ожидаемый результат:**
- Код обучения моделей.
- MLflow эксперименты с полным логированием.
- Model Registry с зарегистрированной моделью.

### 3. Контейнеризация (Модуль 4)

**Задание:**
- Создайте Dockerfile для сервиса инференса (FastAPI или Flask).
- Настройте docker-compose для запуска всех компонентов: API, MLflow, база данных (PostgreSQL), векторная БД (при необходимости).

**Ожидаемый результат:**
- Dockerfile и docker-compose.yml.
- Возможность запуска всего стека командой `docker-compose up`.

### 4. CI/CD и оркестрация (Модуль 5)

**Задание:**
- Настройте CI/CD-пайплайн (GitHub Actions/GitLab CI): линтеры, тесты, сборка образа, деплой на staging.
- Реализуйте DAG в Apache Airflow для автоматического переобучения модели при поступлении новых данных (например, раз в неделю).

**Ожидаемый результат:**
- Файл `.github/workflows/ci-cd.yml`.
- Airflow DAG в файле `dags/retraining.py`.

### 5. Развёртывание и масштабирование (Модуль 6)

**Задание:**
- Разверните REST API для инференса модели (FastAPI).
- Обеспечьте горизонтальное масштабирование (Kubernetes HPA или docker-compose с балансировкой).
- Настройте canary deployment для безопасного обновления модели.

**Ожидаемый результат:**
- Работающий REST API с эндпоинтами `/predict` и `/health`.
- Конфигурация для Kubernetes (deployment.yaml, service.yaml, hpa.yaml).

### 6. Мониторинг (Модуль 7)

**Задание:**
- Настройте сбор метрик: latency, throughput, качество модели (accuracy drift).
- Визуализируйте метрики в Prometheus/Grafana.
- Настройте детекцию дрейфа данных и моделей (Evidently или WhyLogs).
- Настройте алерты при деградации качества.

**Ожидаемый результат:**
- Конфигурация Prometheus и Grafana.
- Дашборды с метриками.
- Отчёт о дрейфе (Evidently).

### 7. Безопасность и Governance (Модуль 8)

**Задание:**
- Настройте управление секретами (Vault/GitHub Secrets).
- Реализуйте RBAC для API (роли: admin, user, readonly).
- Создайте model card с описанием данных, метрик, ограничений и этических аспектов.

**Ожидаемый результат:**
- Файл `model_card.md` с полным описанием.
- Конфигурация RBAC для API.

### 8. Документация

**Задание:**
- Подготовьте README с инструкцией по развёртыванию.
- Опишите архитектуру решения (диаграмма компонентов).
- Подготовьте презентацию (10–15 слайдов) для защиты.

**Ожидаемый результат:**
- README.md.
- Презентация в формате PDF или PPTX.

## Формат сдачи

- Git-репозиторий с кодом, конфигами, DVC-файлами.
- Демонстрация работы сервиса (скриншоты/видео).
- Презентация (10–15 слайдов) с описанием архитектуры, принятых решений, результатов.
- Устная защита с ответами на вопросы.

## Критерии оценивания

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| **Качество данных и EDA** | Данные не подготовлены | EDA выполнено поверхностно | Глубокий анализ, обоснованные решения по очистке |
| **Модели и эксперименты** | Менее 3 моделей или нет MLflow | 3 модели, но трекинг неполный | Полный трекинг, сравнение, обоснованный выбор |
| **Контейнеризация и CI/CD** | Отсутствует | Частичная реализация | Полный стек: Docker, CI/CD, оркестрация |
| **Serving и мониторинг** | Нет API или мониторинга | API есть, мониторинг базовый | Полноценный сервис с мониторингом и алертами |
| **Документация и безопасность** | Документация отсутствует | Документация есть, но неполная | Полная документация, model card, безопасность |
