# Кейс №10. Платформа для генерации и поиска синтетических данных

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Курсовой проект (кейс) |
| **Модули** | 1, 2, 3, 4, 5, 8 |
| **Компетенции** | LC-2, LC-3, LC-5 |
| **Индикаторы** | LC-2.1–2.3, LC-3.1, LC-5.1–5.3, LC-5.5 |
| **Уровень** | П (продвинутый) |
| **Режим выполнения** | Индивидуально |
| **Срок выполнения** | 3 недели |
| **Максимальный балл** | 10 |

## Назначение

Разработать систему генерации синтетических данных для обучения моделей и автоматизированного поиска данных в ИИ-платформе. Проект востребован в ситуациях, когда реальных данных недостаточно или они закрыты по соображениям безопасности.

## Постановка задания

### 1. Подготовка данных (Модуль 2)
- Выберите исходный датасет для генерации синтетических данных.
- Проведите EDA.

### 2. Разработка модели генерации (Модуль 3)
- Реализуйте модель генерации данных (GAN, VAE или диффузионная модель).
- Обучите модель, логируйте эксперименты в MLflow.

### 3. Контейнеризация (Модуль 4)
- Создайте Dockerfile для сервиса генерации.

### 4. CI/CD и оркестрация (Модуль 5)
- Настройте CI/CD для деплоя.

### 5. Безопасность (Модуль 8)
- Обеспечьте контроль качества синтетических данных.
- Создайте документацию по ограничениям.

## Вопросы для защиты

3. Как вы обеспечиваете разнообразие сгенерированных данных?
4. Как вы организовали поиск по сгенерированным данным?
5. Какие ограничения и риски использования синтетических данных вы выявили?
