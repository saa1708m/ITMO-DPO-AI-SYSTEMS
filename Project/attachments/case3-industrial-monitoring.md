# Кейс №3. MLOps-платформа для мониторинга промышленного оборудования

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Курсовой проект (кейс) |
| **Модули** | 1, 2, 4, 5, 6, 7 |
| **Компетенции** | LC-2, LC-3, LC-5 |
| **Индикаторы** | LC-2.1, LC-3.1, LC-5.1–5.4 |
| **Уровень** | С (средний) |
| **Режим выполнения** | Команда 2–3 человека |
| **Срок выполнения** | 3 недели |
| **Максимальный балл** | 10 |

## Назначение

Разработать систему предиктивного мониторинга промышленного оборудования, включающую сбор данных с датчиков, обучение моделей прогнозирования отказов и полный MLOps-цикл. Проект моделирует применение «Сайбокс» в промышленной диагностике. ПАК «Сайбокс» от Т1 позволяет ускорить запуск ИИ-сервисов до 30% и сократить время прототипирования решений до 50%.

## Проверяемые результаты обучения

Студент должен уметь:

- обрабатывать временные ряды с данными датчиков;
- строить модели прогнозирования остаточного ресурса (RUL);
- контейнеризировать сервисы обучения и инференса;
- настраивать автоматическое переобучение моделей;
- организовывать мониторинг дрейфа данных и моделей.

## Постановка задания

### 1. Работа с данными (Модуль 2)

**Задание:**
- Используйте открытый датасет с данными датчиков оборудования (например, NASA Turbofan или аналогичный).
- Проведите EDA: временные ряды, тренды, корреляции.
- Выполните очистку и агрегацию данных.
- Настройте DVC для версионирования.

**Ожидаемый результат:**
- Jupyter Notebook с EDA.
- DVC-репозиторий с данными.

### 2. Разработка моделей (Модуль 3)

**Задание:**
- Реализуйте модели прогнозирования остаточного ресурса (RUL).
- Используйте как минимум 3 подхода: линейная регрессия, Random Forest, LSTM.
- Организуйте эксперименты в MLflow.
- Зарегистрируйте лучшую модель в Model Registry.

**Ожидаемый результат:**
- Код обучения моделей.
- MLflow эксперименты.
- Model Registry с лучшей моделью.

### 3. Контейнеризация (Модуль 4)

**Задание:**
- Создайте Docker-образы для обучения и инференса.
- Настройте docker-compose для локального развёртывания.

**Ожидаемый результат:**
- Dockerfile для обучения и инференса.
- docker-compose.yml.

### 4. Оркестрация (Модуль 5)

**Задание:**
- Реализуйте Airflow DAG для автоматического переобучения модели при поступлении новых данных.
- Настройте CI/CD для деплоя обновлений модели.

**Ожидаемый результат:**
- Airflow DAG в `dags/retraining.py`.
- CI/CD пайплайн (GitHub Actions).

### 5. Serving и мониторинг (Модули 6–7)

**Задание:**
- Разверните REST API для получения прогнозов по данным датчиков.
- Настройте мониторинг дрейфа данных и качества модели (Evidently).
- Визуализируйте метрики в Grafana.

**Ожидаемый результат:**
- Работающий REST API.
- Дашборды в Grafana.
- Отчёт о дрейфе.

## Формат сдачи

- Git-репозиторий с кодом.
- Демонстрация работы системы на тестовых данных.
- Отчёт с описанием архитектуры.

## Критерии оценивания

| Критерий | 0 баллов | 1 балл | 2 балла |
|----------|----------|--------|---------|
| **Обработка временных рядов** | Не выполнена | Базовая обработка | Полный EDA, очистка, агрегация |
| **Модели прогнозирования** | Менее 3 моделей | 3 модели, но без глубокой настройки | 3+ модели с HPO и сравнением |
| **Контейнеризация** | Отсутствует | Частичная | Полный Docker-стек |
| **Оркестрация и CI/CD** | Отсутствует | Частичная | Полный Airflow + CI/CD |
| **Мониторинг** | Отсутствует | Базовый | Полный мониторинг + дашборды |
