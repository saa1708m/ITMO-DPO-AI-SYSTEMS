# Кейс №5. ИИ-агент для автоматизации закупочных процессов

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Курсовой проект (кейс) |
| **Модули** | 1, 2, 3, 4, 5, 8 |
| **Компетенции** | LC-2, LC-3, LC-5 |
| **Индикаторы** | LC-2.1–2.3, LC-3.1, LC-5.1–5.3, LC-5.5 |
| **Уровень** | С (средний) |
| **Режим выполнения** | Индивидуально или в паре |
| **Срок выполнения** | 3 недели |
| **Максимальный балл** | 10 |

## Назначение

Разработать ИИ-агента для автоматизации закупочных процессов: анализ потребностей, поиск поставщиков, проверка документов, формирование заявок. Проект моделирует применение ИИ-решений Т1 для оптимизации закупочной деятельности.

## Постановка задания

### 1. Подготовка данных (Модуль 2)
- Соберите данные о закупках (номенклатура, поставщики, цены, сроки).
- Проведите EDA, очистите данные.
- Настройте DVC.

### 2. Разработка моделей (Модуль 3)
- Реализуйте модель для классификации номенклатуры.
- Реализуйте модель для поиска поставщиков (ранжирование).
- Организуйте эксперименты в MLflow.

### 3. Контейнеризация (Модуль 4)
- Создайте Dockerfile для сервиса инференса.
- Настройте docker-compose.

### 4. CI/CD и оркестрация (Модуль 5)
- Настройте CI/CD для деплоя.
- Реализуйте Airflow DAG для обновления модели.

### 5. Безопасность (Модуль 8)
- Настройте RBAC для API.
- Создайте model card.
