# Кейс №8. Голосовой ИИ-агент для медицинских учреждений

## Паспорт КИМ

| Параметр | Значение |
|----------|----------|
| **Вид КИМ** | Курсовой проект (кейс) |
| **Модули** | 1, 2, 3, 4, 6, 8 |
| **Компетенции** | LC-2, LC-3, LC-5 |
| **Индикаторы** | LC-2.1–2.3, LC-3.1, LC-5.1–5.3, LC-5.5 |
| **Уровень** | С–П (средний–продвинутый) |
| **Режим выполнения** | Индивидуально |
| **Срок выполнения** | 3 недели |
| **Максимальный балл** | 10 |

## Назначение

Разработать голосового ассистента на базе технологий распознавания речи и больших языковых моделей. Аналог решения Т1 для ЦИТО им. Н.Н. Приорова.

## Постановка задания

### 1. Подготовка данных (Модуль 2)
- Соберите аудиозаписи с транскрипциями (можно использовать открытые датасеты).
- Выполните предобработку аудио (нормализация, шумоподавление).

### 2. Разработка моделей (Модуль 3)
- Настройте модель распознавания речи (ASR), например, Whisper или Wav2Vec2.
- Реализуйте модель классификации интентов.
- Настройте LLM для генерации ответов.

### 3. Контейнеризация (Модуль 4)
- Создайте Dockerfile для сервиса.
- Настройте docker-compose.

### 4. Развёртывание API (Модуль 6)
- Создайте REST API для приёма аудио и возврата ответа.

### 5. Безопасность (Модуль 8)
- Обеспечьте безопасность медицинских данных.

3. Как вы организовали генерацию ответов?
4. Как вы обеспечили безопасность медицинских данных?
